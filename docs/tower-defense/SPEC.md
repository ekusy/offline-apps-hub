# タワーディフェンス仕様書

## 1. 概要
画面内の経路に沿って進む敵を、配置したタワーで撃破するリアルタイム戦略ゲーム。HTML5 Canvas で描画。

## 2. 機能要件

### 2.1 マップ
- グリッド 12×9 マスの固定マップ 3 種を内蔵
- 経路（パス）は固定（事前定義）。タワー配置可マスとパスを CSS/座標で区別

### 2.2 ウェーブシステム
- 最大 10 ウェーブ
- ウェーブごとに敵種・敵数・HP・移動速度がスケール
- ウェーブ間は短い準備時間（5 秒、スキップ可）

### 2.3 タワー種別
- 基本タワー（短射程・速射）
- 大砲タワー（広範囲ダメージ・低速）
- 凍結タワー（移動速度低下、ダメージ無）
- 各タワーはコスト/射程/連射/ダメージのバランス調整済み

### 2.4 敵種別
- 通常敵（標準 HP）
- 高速敵（HP 少なめ・速い）
- 装甲敵（HP 高い・遅い）
- ボス（ウェーブ最後に出現）

### 2.5 経済
- 開始ゴールド: 100
- 敵撃破でゴールド獲得
- タワー建設にゴールド消費
- タワー売却で 70% 返却

### 2.6 ゲーム終了
- ライフ 10 → 敵が経路終端到達で減少 → 0 で敗北
- 全ウェーブクリアで勝利

### 2.7 UI
- 上部: ライフ / ゴールド / ウェーブ番号 / 速度切替（1x/2x）
- 下部: タワー選択パレット（コスト表示）
- マップタップ: 建設可マスなら配置、既存タワーなら売却ダイアログ

## 3. 非機能要件
- 60fps を目標、モバイルは 30fps 許容
- ピュア JS、外部ライブラリなし
- localStorage に最高ウェーブ到達数を保存
- 日英対応

## 4. ストレージキー
- `td-best-wave`: 最高到達ウェーブ
- `td-best-stage`: 最高クリアステージ

## 5. アクセスフロー
- ハブから `/apps/tower-defense/` へ
- `apps.js` に登録（カテゴリ: games / サブカテゴリ: strategy）
